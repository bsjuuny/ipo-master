# 💰 IPO Master (공모주 마스터) 개발자 가이드

안녕하세요! 이 프로젝트는 복잡한 **공모주 청약 일정과 정보**를 자동으로 정리해서 사람(학부모, 투자자)이 한눈에 보기 좋게 전달하는 서비스입니다.

---

## 🏗 데이터가 수집되는 과정 (Data Flow)

이 프로젝트는 데이터의 정확성이 생명입니다. 다음 3단계를 통해 매일 아침 업데이트됩니다.

1.  **크롤링**: `scripts/aggregate.ts`가 Finuts나 38커뮤니케이션 같은 사이트에서 청약일, 공모가, 증권사 정보를 긁어옵니다.
2.  **보정 (매우 중요)**: 자동 수집은 가끔 기업 이름이 틀리거나 오타가 날 수 있습니다. 이럴 때 개발자가 `scripts/override.json`에 올바른 정보를 적고 `npm run override:apply`를 실행하면 수동으로 고칠 수 있습니다.
3.  **발송**: 정리가 끝난 데이터는 텔레그램 봇이 "오늘은 OO종목 청약일입니다!" 하고 리포트를 쏩니다.

---

## 🛠 주요 명령어 및 파일 관리

### 1. 데이터 업데이트
- `npm run data:update`: 수집 로직을 돌립니다. (보통 스케줄러가 자동으로 수행)
- `npm run notify:telegram`: 텔레그램으로 요약본을 즉시 발송합니다. 테스트할 때 유용합니다.

### 2. UI 개발 및 배포
- `npm run dev`: 결과물이 나올 대시보드 화면을 수정할 때 사용합니다.
- `src/app/page.tsx`: 메인 화면 소스입니다. 여기서 IPO 리스트 디자인을 바꿀 수 있습니다.

---

## 🧠 핵심 로직 가이드 (수정 시 참고)

### 새로운 수집 사이트를 추가하고 싶다면?
- `scripts/aggregate.ts`에 새로운 크롤링 클래스를 추가해야 합니다. 
- 이 프로젝트는 **Playwright**를 사용하므로, 브라우저에서 버튼을 누르거나 클릭하는 로직을 코드로 짤 수 있습니다.

### 텔레그램 메시지 문구를 바꾸고 싶다면?
- `scripts/generate-ipo-summary.ts`를 찾으세요. 여기서 리포트 제목이나 이모지, 레이아웃을 마음대로 수정할 수 있습니다.

---

## 🚨 문제 해결 (Troubleshooting)

**Q: 사이트에서 데이터를 못 가져와요! (Selection Error)**
- A: 크롤링 대상 사이트의 버튼 이름이나 CSS 클래스가 바뀌었을 가능성이 큽니다. 브라우저 개발자 도구로 바뀐 클래스명을 찾아서 `aggregate.ts`를 업데이트하세요.

**Q: 텔레그램 알림이 중복으로 가요.**
- A: 스케줄러(`c:/github/scheduler`)에서 이 프로젝트를 몇 번 호출하는지 확인하세요. 보통 하루에 한두 번만 가도록 설정되어 있어야 합니다.

---

## 📁 주요 폴더 설명
- `public/data/`: 수집된 IPO 데이터가 `ipo_list.json`이라는 이름으로 저장되는 곳입니다. (이 파일이 앱의 DB 역할을 합니다.)
- `src/types/`: IPO 데이터가 어떤 모양(`id`, `name`, `date` 등)인지 정의된 곳입니다.
