# 프로젝트 통합 문제 해결 및 기술 리포트 (Post-Mortem)

본 문서는 `ipo-master`, `trending-rankings`, `today-signal` 등 모든 프로젝트의 개발 및 운영 과정에서 발생한 주요 장애 케이스와 해결 방법을 정리한 마스터 리포트입니다.

---

## 1. IPO Master 관련 이슈

### 1-1. 데이터 파일(JSON) 오염 및 빌드 실패
- **현황**: `ipo_list.json` 내 Git 충돌 마커 삽입으로 인한 `SyntaxError` 및 빌드 중단.
- **원인**: GHA 워크플로우에서 `git pull` 전 스크래퍼 실행으로 인한 로컬 소스 변경 및 머지 충돌.
- **해결**: 워크플로우 순서 조정(`git pull` 선행) 및 충돌 마커 자동 제거 스크립트 적용.

### 1-2. 텔레그램 알림 시스템 런타임 에러 & 포맷팅 깨짐
- **현황**: 알림 전송 중 `TypeError` 발생 및 메시지에 파워쉘 객체 메타데이터가 섞여 나옴.
- **원인**: 일부 데이터 누락에 대한 예외 처리 미흡 및 파워쉘 환경(`Get-Content`, `ConvertTo-Json`)의 안정성 부족.
- **해결**: 모든 알림 로직을 Node.js(`axios`)로 이전하여 OS 독립적인 안정성 확보. HTML 이스케이프 적용으로 특수 문자 대응.

---

## 2. Trending Rankings 관련 이슈

### 2-1. 텔레그램 한글 깨짐 및 전송 실패
- **현황**: GHA 윈도우 환경에서 파워쉘을 통해 알림 전송 시 한글이 깨지거나 API 호출이 실패함.
- **원인**: 파워쉘의 스크립트 수준 인코딩 처리 한계로 인해 한글 캐릭터가 API 서버에 전달되는 과정에서 손상됨.
- **해결**: 메시지를 노드 스크립트 단계에서 **Base64로 인코딩**하여 전달하고, 파워쉘에서는 이를 디코딩만 하도록 처리하여 인코딩 간섭을 원천 차단.

---

## 3. Today Signal 관련 이슈

### 3-1. GitHub Actions 권한 에러 (403 Forbidden)
- **현황**: 데이터 업데이트 후 `git push` 단계에서 권한 오류로 배포 실패.
- **원인**: GHA 기본 `GITHUB_TOKEN`에 콘텐츠 쓰기(`contents: write`) 권한이 명시적으로 부여되지 않음.
- **해결**: `today-signal` 워크플로우 YAML 파일에 `permissions: contents: write` 설정을 추가하여 해결.

---

## 4. 공통 인프라 (GHA/윈도우 러너) 이슈

### 4-1. 빌드 엔진(tar) 경로 및 캐시 이슈
- **현황**: `npm install` 또는 빌드 과정에서 `tar` 관련 에러 발생 및 캐싱 동작 이상.
- **원인**: 윈도우 환경의 기본 `tar` 경로 충돌 및 npm 캐시 오염으로 인한 종속성 설치 실패.
- **해결**: 워크플로우에서 윈도우 시스템 경로(`C:\Windows\System32`)를 우선순위 상단에 추가하고, 불안정한 npm 캐시(`cache: 'npm'`) 설정을 제거하여 클린 빌드 유도.

---

## 향후 관리 포인트 (Best Practices)
1. **Node.js 중심 개발**: 윈도우 서버 환경 변수를 최소화하기 위해 가급적 모든 로직(알림, 데이터 처리 등)은 Node.js 런타임 내부에서 완결합니다.
2. **강력한 방어 로직**: 데이터가 없거나 형식이 불완전할 경우를 대비한 `null/undefined` 체크를 필수로 수행합니다.
3. **권한 및 보안**: GHA 사용 시 최소 권한 원칙을 지키되, 데이터 쓰기가 필요한 경우 명시적으로 권한을 선언합니다.
4. **인코딩 표준화**: 한글 등 멀티바이트 문자 처리 시 쉘의 인코딩 설정에 의존하지 않고 Base64 등을 활용한 안전한 전달 방식을 선호합니다.
